# リサーチレポート：v3型別素材収集 (2026-04-21)

## サマリー
ADHD当事者の共感獲得コンテンツは「自分たちの矛盾する行動パターンの言語化」と「AIとの相性の発見」に集中している。特に「完璧主義なのに先延ばし」「いい人を演じて自壊」「やることがあると別のことをする」といった自虐的な特性描写が3万超のいいねを稼いでいる。一方、重度ADHD当事者によるAI実装報告（Claude Code・GPTへの病歴学習）は5700いいね程度だが、未開拓の実装ノウハウ領域として機能している。

## 主要トレンド

### 1. 「矛盾する行動パターン」の言語化が最強の共感装置
- **D-0.3での徹夜勉強で成功** (いいね34,475 / RT5,573) ：スケジュール感覚の崩壊と結果の不安定性を自虐的に語る
- **「完璧主義なのに先延ばし」** (いいね17,607 / RT2,027) ：一度走ると強いが、スタート前のエネルギー消耗という二項対立の提示
- **「やることがあると逆に別のことをする」** (いいね6,481 / RT676) ：締め切り・勉強・仕事という具体的状況での行動矛盾

**共通点**：根拠なき励ましではなく「お前らこれだよな」という現象の正確な描写

### 2. 「いい人演技の自壊プロセス」が重度当事者の深刻な悩み
- **「普通のふりをすること」による過剰疲労** (いいね10,748 / RT1,423 ×2件重複) ：空気読み試行による脳リソース枯渇
- **「いい人を演じ続けると自ら壊す」** (いいね6,888 / RT632) ：人間関係リセット・ドタキャン・わざと嫌われる行動の言語化

**未対応ニーズ**：この自壊ループを「構造的に」止める具体策がない

### 3. AIと個人情報の深い結合が実装段階へ
- **重度ADHD当事者のAI学習報告** (いいね5,704 / RT371 ×2件) ：GPT・Gemini・Claude Codeに「病歴・仕事略歴・スケジュール・生い立ち」を学習させた継続的タスク管理
- **Claude への「キャラクター設定」による先延ばし削減** (いいね5,857 / RT1,755) ：AIに役割を与えることで行動開始の抵抗感低減

**注目点**：単なる「AIツール活用」ではなく、AI=外部脳化への移行段階。ただし実装詳細が公開されていない

### 4. 初期評価と衰退のギャップ：「新しい環境での期待値と現実のズレ」
- **言語能力・愛想・アイデア→期待値上昇** (いいね17,594 / RT2,050 ×5件重複) ：初期の高評価と「だんだんと上手くいかない」推移を示唆

**未言語化エリア**：なぜ半年目に失速するのか、その構造的理由

### 5. 「許容度の高さ」による環境適応の罪
- **スマホ破損・電球未交換を放置** (いいね21,076 / RT2,045) ：「困るまで動かない」が実は認知負荷軽減戦略だが、周囲には怠慢に見える

**課題**：この適応戦略が社会的評価を落とすパラドックスが未解決

## 市場の穴（機会）

### 1. 「いい人演技の自壊ループ」を止める構造設計
**現状**：「自分を好きになろう」系の精神論か「人付き合いを減らせ」の消極策のみ

**未開拓領域**：
- 職場で「いい人」を演じることなく、ADHD特性を組織的に活かす配置事例
- Slackのステータス設定・メール自動返信・定期的な「作業中断通知」など、**環境側の仕組みで「いい人」を強制する**具体例
- 「自壊を選ぶ理由」の深掘り（実は人間関係の過剰負荷から逃げるサバイバル戦術では？）

### 2. AI学習の「安全性と実装化」ギャップ
**現状**：「Claude Codeに全情報学習させた」という報告のみ。実装方法・プロンプト・コスト・プライバシー懸念が空白

**未開拓領域**：
- 個人情報を与えず、AIが「ADHD脳の意思決定パターン」を学習するプロンプト設計の公開
- 「キャラクター設定で先延ばし削減」の詳細再現ステップ
- 重度当事者が月いくら払ってどのAIを使ってるのか、コスト対効果のリアル

### 3. 「初期高評価→半年で失速」のプロセス解析
**現状**：「期待値が上がるから失速する」は言及されるが、その時間軸（3ヶ月？6ヶ月？）と具体的な失速パターン（ミス増加？提出遅延？コミュニケーション劣化？）が未整理

**未開拓領域**：
- 組織内での「ADHD的成功曲線」の可視化レポート
- 失速時点での具体的な困り事（→構造的対策の足がかり）
- 新人ADHD×古参ADHD の対比（ベテランは工夫で何を乗り切ってるか）

## データ補足

### 限界・注意点
1. **Xデータのみ**：韓国語含むため全文翻訳できず、微細なニュアンス喪失の可能性（特に「D-0.3」の成功報告）
2. **RTクラスタリング不明**：同一ユーザーによる拡散か、複数セグメントへの自然拡散かが不分明
3. **インプレッション数未収集**：いいねRTが高い＝拡散性が高いが、実ターゲット到達範囲は未測定
4. **note・ブログ記事データなし**：「共感系」のコンテンツ形式がX投稿に偏っている可能性（長文での掘り下げニーズが隠れている可能性）
5. **AI実装報告の詳細不足**：「Claude Codeに学習させた」の具体的プロンプト・コスト・継続率が記載なし（実装再現性が低い）

### データ品質スコア
- **共感・あるあるテーマの信頼度**：★★★★★（3万超いいね、複数言語での同一テーマ）
- **AI実装報告の具体度**：★★☆☆☆（導入決定は高いが再現性不足）
- **職種別成功事例の深さ**：★★☆☆☆（「成功してる」は言及されるが職業名・環境が未記載）
